## Tasks

### Task 1: Create src/components/FingeringPatternControls.test.tsx

Write 6 tests using Vitest + React Testing Library. Match the existing test file style in the repo (check src/*.test.tsx for patterns). Use `vi.fn()` for all callback props.

Tests:
1. `renders all fingering pattern options` — render with fingeringPattern='all', assert buttons "All", "CAGED", "3NPS" are present
2. `calls setFingeringPattern on button click` — click "CAGED" button, assert setFingeringPattern called with 'caged'
3. `shows Shape and Shape Labels sections only when fingeringPattern === "caged"` — render with fingeringPattern='caged', assert "Shape" label visible; render with fingeringPattern='all', assert "Shape" label absent
4. `shows Position section only when fingeringPattern === "3nps"` — render with fingeringPattern='3nps', assert "Position" label visible; render with fingeringPattern='all', assert absent
5. `handles shift-click multi-select for CAGED shapes` — render with fingeringPattern='caged', cagedShapes=new Set(['C']), simulate shift+click on 'A' button, assert setCagedShapes called with Set containing both 'C' and 'A'
6. `calls setDisplayFormat when Note Labels button clicked` — click "Intervals" button, assert setDisplayFormat called with 'degrees'

Provide all required props as `vi.fn()` stubs with sensible defaults. Use `renderWithJotai` wrapper if it exists in the test helpers; otherwise use plain `render` from `@testing-library/react`.

Commit: `test(FingeringPatternControls): add unit tests for fingering pattern controls` — or combine with Task 2 in a single commit (`test(components): add unit tests for FingeringPatternControls and ScaleChordControls`) when both test files are written together, since they test components from the same extraction phase.

### Task 2: Create src/components/ScaleChordControls.test.tsx

Write 6 tests using Vitest + React Testing Library. Provide DrawerSelector-compatible props (scaleOptions, chordOptions, chordFilterOptions as string arrays).

Tests:
1. `renders Scale and Chord Overlay drawers` — render with chordType=null, assert "Scale" and "Chord Overlay" labels present
2. `shows chord-specific controls only when chordType is set` — render with chordType=null, assert "Link chord root to scale" absent; render with chordType='maj', assert it is present
3. `renders NoteGrid when linkChordRoot is false` — render with chordType='maj', linkChordRoot=false, assert note-grid div present
4. `hides NoteGrid when linkChordRoot is true` — render with chordType='maj', linkChordRoot=true, assert note-grid div absent
5. `links chord root to scale root on chordType selection if linkChordRoot enabled` — render with linkChordRoot=true, rootNote='D', simulate selecting a chordType, assert setChordRoot called with 'D'
6. `renders Interval Filter drawer when chordType is set` — render with chordType='maj', assert "Interval Filter" label present

Use vi.fn() for all setter props. If DrawerSelector requires complex DOM setup, render the full component tree (React Testing Library renders full DOM).

Commit: `test(ScaleChordControls): add unit tests for scale and chord controls` — may be omitted if both test files are committed together with Task 1.
