## Objective

Split the monolithic CSS into four layers: tokens.css (design tokens), index.css (global resets only), src/components/shared.css (reusable UI patterns), and component-specific CSS stubs. This creates the CSS foundation that extracted components in wave 2 will import.

- @.vbw-planning/phases/01-extract-duplicated-controls/.context-lead.md — phase scope and decisions

## Tasks

### Task 1: Create src/tokens.css

Read `src/index.css` lines 1–79. Extract the entire `:root { }` block (all CSS custom properties: colors, spacing, typography, transitions, CAGED colors, radius values, shadow values) into a new file `src/tokens.css`. The file contains only this `:root` block. Import `tokens.css` in `src/main.tsx` as the first import (before the existing `index.css` import).

Commit: `style(tokens): extract CSS custom properties to tokens.css`

### Task 2: Refactor src/index.css to resets only

Remove the `:root { }` block from `src/index.css` (now that it lives in tokens.css). Keep all global reset rules: `*`, `body`, `#root`, `::-webkit-scrollbar`, and related selectors. Result: index.css is ~50 lines of reset-only rules. Verify no CSS variable references break by checking that tokens.css is imported before index.css in main.tsx.

Commit: `style(index): remove :root block, keep global resets only`

### Task 3: Create src/components/shared.css

Create `src/components/shared.css`. Move the following rule sets from `src/App.css` into shared.css (preserving exact CSS rules, comments, and selector specificity):
- `.toggle-group` and `.toggle-btn` (all states: hover, active, focus-visible, .active modifier)
- `.control-section` and `.section-label`
- `.note-grid` and `.note-btn` (all states: hover, .active modifier)
- `.chord-root-row`
- `.link-toggle` and `.link-toggle input[type="checkbox"]`
- `.accidental-toggle` and related selectors

After moving, delete those rule sets from `src/App.css`. The `src/components/` directory must exist (create it if needed — it will also hold component TSX files from wave 2 plans).

Commit: `style(shared): create shared.css with reusable UI patterns extracted from App.css`

### Task 4: Verify build and tests pass

Run `npm run build` — must succeed. Run `npm run test` — all tests must pass. If any snapshot test fails due to CSS import changes (unlikely since CSS changes don't affect DOM structure), update snapshots with `npm run test -- --update-snapshots` and commit the snapshot diff. Confirm that removing shared classes from App.css did not break any styles by checking that the moved selectors still exist (now in shared.css).

Commit: `test(css): verify build and tests pass after CSS restructuring` — only if snapshot updates are needed; otherwise no commit for this task.
