## Design notes (frontend-design)

The FretFlow aesthetic stays: dark faceplate, neon-cyan accents, mono numerals, uppercase micro-labels with `letter-spacing: 0.12em`. Specific commitments for Plan G:

- **`fit` correctness.** `.labeled-select` keeps its flex-column layout (label-above-trigger). The trigger gets `align-self: flex-start` when `data-fit` is set so it sizes to content. Min-width default `5rem` is honored. The Voicing/Quality/Root/Pattern/StringSet/TimeSignature triggers all collapse to ~4‚Äì9rem instead of stretching.
- **Voicing row alignment.** Restore an inspector-style micro-label above the Voicing dropdown ("VOICING") so the row reads `VOICING | LENS | STRINGS | LOCK TO SCALE` with consistent label height ‚Äî every visible cell in the CHORD row has the same baseline.
- **Rename "Snap to scale" ‚Üí "Lock to scale window".** "Snap" implied a pull-into-position behavior; "lock" describes the actual gating semantics. The i18n key becomes `inspector.chordLockToScaleLabel`; storage atom key stays the same so user preference persists across the rename.
- **Tabs as tabs.** Drop the `border-radius: 999px` + outlined-pill chrome from `.root[data-placement="top"] .tab`. Keep the baseline `.tab` styling (transparent, 2px transparent bottom border) and let the active state's accent-colored bottom border do the indicator work. The `.tabHeader` already has a `border-bottom: 1px solid var(--faceplate-divider)` ‚Äî the active tab's 2px accent border sits cleanly on that baseline.
- **Shared string-set primitive.** Lift `ChordStringSetPicker`'s logic into a generic `StringSetPicker` in `src/components/shared/` that takes `{ value, onChange, options, label }` and renders the same LabeledSelect chrome. The chord variant, the 1-string variant, and the 2-string variant all wrap it. Label format stays the guitar-numbered "1¬∑2¬∑3¬∑4" ‚Äî the dot separator is unique enough to avoid confusion with the "Strings 1-2" range syntax (which we retire from FingeringPatternControls).
- **Disabled options inside the string-set dropdown.** When snap is on and a string set has no candidate voicings in the active scale window, the option renders disabled with a `title` tooltip ("No voicing in the C-shape window"). The user sees the option exists but can't pick it ‚Äî this is more discoverable than silently omitting it.
- **CAGED octave fix.** The `cagedShapeDataAtom` (or whichever atom feeds `<FretboardSVG>`'s polygon layer) reads `cagedOctaveAtom` when a single shape is active and filters `polygons[]` / `coordinates[]` / `bounds[]` to the chosen octave instance. When `cagedShapes.size > 1` the octave atom is ignored (and the toggle is already hidden by FingeringPatternControls' existing `cagedShapes.size === 1` gate). The scale-window atom keeps its existing `cagedOctaveAtom` read so chord-window snapping stays consistent.
- **Outside-chord handling, end-to-end.** The investigation revealed a deeper bug: `resolveProgressionStep` (the resolver that turns `{ degree, manualRoot, qualityOverride }` ‚Üí concrete root/quality/label) **never reads `manualRoot`**. Only `activeChordRootAtom` in `songStateAtoms.ts` works around it via a short-circuit, which is why the chord overlay updates on a borrowed pick but the always-on `ProgressionTrack` timeline (and the editor-pane pill) keep showing the diatonic chord. The fix is in the resolver itself: when `manualRoot` is set, derive the chord from it (default quality = the most-likely triad on that root within the active scale, or the prior `qualityOverride` if the user already set one). Once the resolver honors `manualRoot`, the `activeChordRootAtom` short-circuit becomes dead code and gets deleted. The Song-tab editor-cell also clears `qualityOverride` on borrowed select so the new root gets a fresh default (instead of inheriting a quality that no longer makes sense).
- **Song-tab progression list is redundant.** `ProgressionTrack` is always rendered above the fretboard, displays the same `resolvedProgressionStepsAtom` data, and is already clickable to set the active step. The Song-tab's inline `step-list` (SongControls.tsx lines 220-240) repeats this affordance in a different visual format. Delete the list. The Song tab then collapses to a single editor pane spanning the full row, eliminating the left/right split that was eating horizontal space.
- **Status bar full-bleed.** Drop the `0.85rem` horizontal padding on `.status-bar`. The label/value pairs already have internal gap and reach the screen edges cleanly.

### Task G4 ‚Äî Fix snap-to-scale to gate by actual scale position keys

**Files:**
- Modify: `src/store/chordOverlayAtoms.ts`
- Test: `src/store/chordOverlayAtoms.test.ts`

The current bug: `closeCandidatesAtom` filters voicings by checking whether `min(frettedFrets) >= scaleWindow.lo` and `max(frettedFrets) <= scaleWindow.hi`. A voicing whose notes sit on strings the scale pattern doesn't include (or on frets within the band but not on the pattern's actual lattice) still passes. Fix: collect the actual set of position keys covered by the active scale pattern and require each voicing's `positionKeys` to be a subset (open strings excepted, since they don't anchor to a pattern fret).

Common candidates: `activeScalePositionsAtom`, `cagedShapeDataAtom.coordinates`, `npsShapeDataAtom.coordinates`. Confirm the actual name. If no single atom unions them, derive one ‚Äî `activeScalePatternPositionsAtom: atom<Set<string>>`.

```tsx
// src/components/shared/StringSetPicker.tsx
import { useMemo } from "react";
import { LabeledSelect } from "../LabeledSelect/LabeledSelect";

export interface StringSetOptionInput {
  /** Stable id, used as the LabeledSelect value. */
  id: string;
  /** Optional explicit label. If omitted, derived from `strings`. */
  label?: string;
  /** Optional 0-based string indices (high E = 0). Used to format the label
   * when `label` is not provided ‚Äî produces "1¬∑2¬∑3¬∑4" (guitar-numbered). */
  strings?: readonly number[];
  /** When true, the option is rendered disabled. Pair with `disabledReason`. */
  disabled?: boolean;
  /** Tooltip text shown when hovering a disabled option. */
  disabledReason?: string;
}

export interface StringSetPickerProps {
  label: string;
  value: string;
  onChange: (value: string) => void;
  options: readonly StringSetOptionInput[];
  /** Override the i18n "All" label when `id === "all"` and no label given. */
  allLabel?: string;
}

function formatLabel(opt: StringSetOptionInput, allLabel: string): string {
  if (opt.label) return opt.label;
  if (opt.id === "all") return allLabel;
```
