# v2.0 Plan I Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close 5 follow-on issues from Plan H — remove the CAGED Octave feature, fix the Pattern select width, make the status bar truly full-bleed, restructure the Inspector layout so groups can sit side-by-side (and fix the Duration/Quality gap), and rework the Lock-to-scale close-voicing filter to a "majority fits" semantic with auto-fallback to the next enabled string set.

**Architecture:** Three changes are mechanical (CAGED Octave removal, Pattern width, status bar escape). Two are architectural pivots:
1. **Layout** — keep CSS Grid for intra-group control rows (it's the right tool), introduce an outer flex composer so multiple groups can wrap into 2 columns at wide widths. Replace the editor-pane's `1fr / auto` grid with a content-sized flex row so Duration + Quality sit together.
2. **Lock-to-scale filter** — replace the binary ALL-fit predicate (Plan H-T5) with a majority-fit predicate (`>= ceil(fretted × 2/3)` notes within the active scale window). The same predicate powers both the close-candidate filter and the per-string-set disabled probe. Add auto-fallback in `effectiveStringSetAtom` so a disabled picked stringset gracefully resolves to the first enabled one.

**Tech Stack:** React 19 + Jotai + TypeScript, Radix UI Select, CSS Modules, Tonal.js via `@fretflow/core`, Vitest + Playwright.

**Branch:** `claude/elated-nobel-dd4e76` (extends PR #451 — keeps the `feat(v2.0)!:` breaking marker; merging triggers `v2.0.0` via Auto Release).

### Task I-T6: Lock-to-scale uses majority-fit predicate

**Files:**
- Modify: `src/store/chordOverlayAtoms.ts`
- Test: `src/store/chordOverlayAtoms.test.ts`

Plan H-T5 reverted the snap filter to ALL-fit (`Math.min(fretted) >= lo && Math.max(fretted) <= hi`). The user wants a softer rule: a voicing passes if MOST of its fretted notes lie within `[lo, hi]`. "Most" = `ceil(frettedCount × 2/3)`.

Extract the predicate into a helper for clarity and reuse:

```ts
const FIT_RATIO_NUMERATOR = 2;
const FIT_RATIO_DENOMINATOR = 3;

function majorityFitsInWindow(
  positionKeys: readonly string[],
  window: { lo: number; hi: number },
): boolean {
  const fretted = positionKeys
    .map((k) => Number(k.split("-")[1] ?? 0))
    .filter((f) => f > 0);
  if (fretted.length === 0) return true; // all-open voicings always fit
  const threshold = Math.ceil(
    (fretted.length * FIT_RATIO_NUMERATOR) / FIT_RATIO_DENOMINATOR,
  );
  const inside = fretted.filter((f) => f >= window.lo && f <= window.hi).length;
  return inside >= threshold;
}
```

Apply to BOTH `closeCandidatesAtom` and `closeCandidatesAllStringSetsAtom`.

const SCALE_FIT_NUMERATOR = 2;
const SCALE_FIT_DENOMINATOR = 3;

/** A voicing "majority-fits" the scale window when at least ceil(fretted × 2/3)
 * of its fretted notes lie within `[window.lo, window.hi]`. Open strings are
 * not bound by the window (they're playable at any hand position). */
function majorityFitsInWindow(
  positionKeys: readonly string[],
  window: { lo: number; hi: number },
): boolean {
  const fretted = positionKeys
    .map((k) => Number(k.split("-")[1] ?? 0))
    .filter((f) => f > 0);
  if (fretted.length === 0) return true;
  const threshold = Math.ceil(
    (fretted.length * SCALE_FIT_NUMERATOR) / SCALE_FIT_DENOMINATOR,
  );
  const inside = fretted.filter((f) => f >= window.lo && f <= window.hi).length;
  return inside >= threshold;
}
